覺得在 Git Bash 這樣子黑黑的視窗中執行各種 Git 指令非常地酷！
有如電影與動漫中常見的電腦高手在緊急時刻控制全局時所會操作的介面，
彷彿自己也變得厲害許多。
但這種全都要靠自己手動輸入指令的操作也著實非常地有難度。

在日常中我們已經很習慣有精美且易操作的 GUI 來幫助我們使用這些技術與工具，
因為有了GitHub、GitLab這樣的平台，我們不再需要特別去記憶 Git 指令，而是可以很輕鬆地依靠點擊執行每一個動作，
如今要抽換習以為常的操作手法，轉而在極素地視窗中控制檔案的存取，也是頻頻遇到各種障礙。

當然最大的麻煩不在於查找/輸入指令，
而在於「自己對於版本控制方面的知識與邏輯非常地不了解」，「實作到大型專案且使用版本控制的機會也更是幾乎沒有」，
使得即使武器已握於手中，卻不知如何痛打敵人的弱點，
(懂得有哪些指令可以下，但不懂得使用時機與邏輯)
這是我認為當前的我所適逢，最大的困難。
